Comments

Single-line Comments

Use the hash symbol (#) for single-line comments:


x = 5 # This sets x to 5
# This entire line is a comment
result = x + 10  # Add 10 to x

Multi-line Comments

Use double hash symbols (##) for multi-line comments:


##
This is a multi-line comment
It can span multiple lines
The interpreter ignores everything between the markers
##

x = 42  # Code continues here

Comment Rules

  • Comments cannot be nested
  • Multi-line comment markers must be on their own lines (whitespace allowed)
  • Single-line comments can appear anywhere on a line
  • Code after a comment marker on the same line is ignored
  • Comments do not affect line numbering for error reporting

Best Practices

  • Use comments to explain complex logic
  • Keep comments up-to-date with code changes
  • Use multi-line comments for temporary code disabling
  • Add meaningful comments rather than stating the obvious
